Introduction

This is the first Universal Windows sample app for the Microsoft Band targetting VS2015.

The Microsoft Band SDK comes with a sample app for VS2013.

VS2015, Windows 10 and the Windows 10 SDK have been released so this is to start you going with the latest development tools and the Universal Windows Platform.

This sample takes readings from all the sensors exposed by the Microsoft Band SDK for your convenience.

Learn by example and save one hour fiddling around!

Building the Sample

This sample requires Visual Studio 2015 RTM Community edition: yes the free version.

The project requires the NuGet Package : Microsoft Band SDK (should be automatically refreshed)

 The Windows 10 SDK : should be distributed with all VS2015 editions.

You will need a Microsoft Band bluetooth-paired to your development machine or targetted device.

There are no Microsoft Band emulators at this time...


Description

This sample shows that the Microsoft Band SDK that originally targetted VS2013 does work fine with VS2015.

To reproduce this project from scratch:

  • Start with a new blank Universal Windows App project
  • Use NuGet Package Manager to find and install the latest Microsoft Band SDK
  • Manually update the app manifest with the correct bluetooth permissions as the designer does not yet support the required settings.
  • Add code to connect to the Band, subscribe to the sensors' data and take samples for 10 seconds.
I will publish a blog post to compliment this sample and update this description to the step-by-step guide as soon as it is published.

If you run the app without first bluetooth-pairing the Band to the device running the sample app you will receive an error. To pair the devices, on the Band go to settings, bluetooth, pairing. Go to the device that will run the UWA and find your Band in the bluetooth settings.

This app samples all the sensors currently available in the Microsoft Band SDK:

  • Accelerometer
  • Calories
  • Contact
  • Distance
  • Gyroscope
  • HeartRate
  • Pedometer
  • SkinTemperature
  • UV

There are hundreds of accelerometer and gyroscope readings per second so the app only displays every 20th reading so as not to overwhelm the UI with updates.

The app connects to your Microsoft Band and attempts to take readings for 10 seconds when you click the 'Run' button.

After the 10 second sample period, the app deletes the sensor subscriptions and displays a summary of the number of readings received from each sensor.

                    int samplesReceivedAcc = 0; // the number of Accelerometer samples received
                    // etc..

                    // Subscribe to Accelerometer data.
                    bandClient.SensorManager.Accelerometer.ReadingChanged += (s, args) =>
                    {
                        samplesReceivedAcc++;
                        if ((samplesReceivedAcc % 20) == 0)
                        {
                            // Only report occasional Accelerometer data
                            IBandAccelerometerReading readings = args.SensorReading;
                            CoreApplication.MainView.CoreWindow.Dispatcher.RunAsync(CoreDispatcherPriority.Normal, () =>
                            {
                                this.txtAccelerometer.Text = readings.AccelerationX.ToString() + " " + readings.AccelerationY.ToString() + " " + readings.AccelerationZ.ToString();
                            });
                        }
                    };
                    await bandClient.SensorManager.Accelerometer.StartReadingsAsync();

                    bandClient.SensorManager.Calories.ReadingChanged += (s, args) =>
                    {
                        samplesReceivedCal++;

                        {
                            this.txtCalories.Text = readings.Calories.ToString();
                        });
                    };
